ИНСТРУКЦИЯ: как залить готовый Firebase-сайт в GitHub Pages
===========================================================

Что делает этот комплект
------------------------
Он заменяет локальное хранение данных на общую базу Firebase Firestore.
После этого:
- добавленные OP/ED будут видны всем;
- оценки будут общими;
- каждый пользователь выбирает ник;
- можно переоценивать треки;
- вкладка «Сезоны опенингов» показывает годы до 1990;
- с сентября автоматически появится следующий год;
- зелёная галочка/✅ появляется у сезона, если с твоего ника оценены все треки сезона.

Файлы в комплекте
-----------------
1. index.html
   Главная страница сайта. Её нужно положить в корень репозитория вместо старой.

2. oped-app.js
   Вся логика сайта: чтение из Firebase, оценки, сезоны, профиль, добавление треков.

3. firebase-config.js
   Файл с настройками Firebase. В нём нужно заменить заглушки на свои значения.

4. firestore.rules
   Правила безопасности Firestore. Их нужно скопировать в Firebase Console.

5. firebase-import.html
   Разовая страница для импорта стартовых данных из Excel-таблицы.

6. firebase_import_helper.js
   Помощник для импорта. Нужен странице firebase-import.html.

7. op_ed_initial_data.js
   Готовые стартовые данные:
   - 2094 опенинга;
   - 8366 оценок.

8. op_ed_initial_data.json
   То же самое в JSON-формате. Для сайта не обязателен, но оставлен на всякий случай.


ШАГ 1. Создать проект Firebase
------------------------------
1. Открой https://console.firebase.google.com/
2. Нажми Add project / Создать проект.
3. Название любое, например op-ed-orden.
4. Google Analytics можно выключить.
5. Дождись создания проекта.


ШАГ 2. Создать Web App и взять firebaseConfig
---------------------------------------------
1. Внутри проекта Firebase нажми значок Web: </>
2. Название приложения любое, например op-ed-orden-site.
3. Firebase покажет код вида:

   const firebaseConfig = {
     apiKey: "...",
     authDomain: "...",
     projectId: "...",
     storageBucket: "...",
     messagingSenderId: "...",
     appId: "..."
   };

4. Открой файл firebase-config.js из этого комплекта.
5. Замени заглушки ВСТАВЬ_... на реальные значения из Firebase.
6. Сохрани файл.

ВАЖНО:
firebaseConfig — это не пароль. Его нормально хранить в коде сайта.
Но правила Firestore всё равно обязательно настроить.


ШАГ 3. Включить Anonymous Auth
------------------------------
1. В Firebase Console открой Authentication.
2. Перейди в Sign-in method / Метод входа.
3. Найди Anonymous / Анонимный.
4. Включи его.
5. Сохрани.

Без этого сайт не сможет записывать оценки и новые треки.


ШАГ 4. Создать Firestore Database
---------------------------------
1. В Firebase Console открой Firestore Database.
2. Нажми Create database.
3. Выбери Production mode.
4. Регион выбери любой европейский, если есть выбор.
5. Создай базу.


ШАГ 5. Вставить правила Firestore
---------------------------------
1. В Firebase Console открой Firestore Database.
2. Перейди во вкладку Rules.
3. Открой файл firestore.rules из этого комплекта.
4. Полностью скопируй его содержимое.
5. Вставь в Rules вместо старого текста.
6. Нажми Publish.


ШАГ 6. Загрузить файлы в GitHub
-------------------------------
Есть два варианта.

Вариант А — через сайт GitHub, проще:

1. Открой репозиторий:
   https://github.com/antonanatolevic21-del/op-ed-orden

2. Нажми Add file.

3. Нажми Upload files.

4. Перетащи в окно GitHub эти файлы:
   - index.html
   - oped-app.js
   - firebase-config.js
   - firestore.rules
   - firebase-import.html
   - firebase_import_helper.js
   - op_ed_initial_data.js
   - op_ed_initial_data.json
   - README_ЗАГРУЗКА_В_GITHUB_ПО_ШАГАМ.txt

5. GitHub спросит про замену index.html — это нормально.

6. Внизу напиши commit message, например:
   Firebase Firestore shared data

7. Нажми Commit changes.

Вариант Б — через Git на компьютере:

1. Скачай/клонируй репозиторий.
2. Скопируй файлы из комплекта в папку репозитория.
3. Выполни:

   git add .
   git commit -m "Firebase Firestore shared data"
   git push


ШАГ 7. Подождать публикацию GitHub Pages
----------------------------------------
Обычно сайт обновляется не мгновенно.
Подожди примерно минуту и открой:

https://antonanatolevic21-del.github.io/op-ed-orden/

Если видишь предупреждение про Firebase config — значит в firebase-config.js остались заглушки.


ШАГ 8. Импортировать стартовые данные
-------------------------------------
Когда сайт открылся и Firebase подключён:

1. Открой страницу:
   https://antonanatolevic21-del.github.io/op-ed-orden/firebase-import.html

2. Нажми «Начать импорт».

3. Дождись сообщения:
   ГОТОВО: 2094 опенингов, 8366 оценок.

4. Открой главную страницу:
   https://antonanatolevic21-del.github.io/op-ed-orden/

5. Проверь, что появились треки и оценки.


ШАГ 9. После импорта
--------------------
Лучше удалить из репозитория файл:

firebase-import.html

Почему:
- он уже не нужен каждый день;
- так меньше шансов, что кто-то случайно нажмёт импорт ещё раз.

Повторный импорт в этой версии не должен создать дубли стартовых опенингов,
потому что id берётся из tempId, но всё равно лучше не держать лишнюю админ-страницу публично.

Файлы op_ed_initial_data.js и firebase_import_helper.js после импорта тоже можно удалить,
если больше не планируешь импортировать данные.


Как проверить, что всё работает
-------------------------------
1. Открой сайт в обычном браузере.
2. Выбери ник.
3. Поставь оценку любому треку.
4. Открой сайт в другом браузере или на телефоне.
5. Выбери другой ник.
6. Проверь, что общий рейтинг изменился.
7. Добавь новый трек через «+ добавить трек».
8. Проверь, что он появился на другом устройстве.


Если что-то сломалось
---------------------
1. Если сайт пишет «Firebase не подключён»:
   проверь firebase-config.js.

2. Если оценки не сохраняются:
   проверь, включён ли Anonymous Auth.

3. Если пишет permission-denied:
   проверь, опубликованы ли правила из firestore.rules.

4. Если после загрузки файлов сайт всё ещё старый:
   подожди минуту, обнови страницу Ctrl+F5.

5. Если импорт завис:
   открой консоль браузера F12 -> Console и посмотри ошибку.
